# Соглашения по backend-коду

Эта страница фиксирует наблюдаемые в коде соглашения и рекомендуемые правила консистентности.

## Текущие соглашения

- Роутеры FastAPI сгруппированы по доменам (`auth`, `habits`, `profile`, `calendar`, `notifications`).
- Pydantic-модели задают request/response-контракты в `schemas.py`.
- SQLAlchemy-модели централизованы в `models.py`.
- Зависимости роутов оформлены через `typing.Annotated` и `Depends`.
- Security-хелперы (`hash_password`, `decode_token`, генерация токенов) централизованы.

## Рекомендуемые соглашения

- Оставлять в роутерах только транспортный слой; нетривиальные правила выносить в сервисы.
- Для операций с несколькими записями использовать сервисные функции, чтобы централизовать инварианты и транзакционный контекст.
- Явно задавать response-модели у эндпоинтов, чтобы избежать дрейфа контрактов.
- Логику таймзон и recurrence держать только в сервисных модулях.
- Добавить структурированные коды ошибок для стабильной обработки на frontend.

## Именование и структура

- Путь и префикс роутера должны соответствовать доменному существительному.
- Имена сервисных модулей должны отражать назначение (`notification_worker`, `calendar_agg`).
- Утилиты по возможности держать чистыми (без side effects в БД), если иное не оговорено явно.
