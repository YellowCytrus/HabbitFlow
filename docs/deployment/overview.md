# Обзор деплоя

HabitFlow разворачивается через Docker Compose и опциональный VPS-bootstrap скрипт.

## Топология compose

- `db`: PostgreSQL 16, постоянный volume, внутренняя Docker-сеть.
- `api`: контейнер FastAPI, опубликован на порту `8000`, выполняет миграции в startup-последовательности.
- `web`: сборка frontend под Nginx, опубликована на `5173`.
- опциональный профиль `pgadmin` для контролируемого доступа к БД.

## Последовательность запуска

1. Запуск базы данных и ожидание health.
2. Прогон Alembic-миграций.
3. Запуск API-сервиса.
4. Запуск frontend-сервиса.

## Автоматизация VPS

- `deploy.sh`: one-shot деплой с ожиданием health и выполнением миграций.
- `scripts/vps-bootstrap.sh`: bootstrap хоста (Docker, UFW, Caddy, инициализация `.env`).
- Caddy терминирует HTTPS и проксирует трафик к frontend-контейнеру.

## Операционные риски

- Startup-команда API-контейнера сейчас включает запуск seed-скрипта.
- Логика notification worker не развернута отдельным сервисом, поэтому надежность напоминаний зависит от браузера.
- Rollback-поведение миграций ограничено для metadata-sync ревизий.
