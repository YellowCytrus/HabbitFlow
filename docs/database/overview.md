# Обзор базы данных

Движок БД: PostgreSQL. ORM: SQLAlchemy 2 (`Mapped`, `mapped_column`). Миграции: Alembic.

## Ключевые сущности

- `users`: учетная запись пользователя, auth-флаги, OAuth-привязки.
- `pending_registrations`: промежуточное состояние регистрации до подтверждения кода.
- `habits`: привычки пользователя с recurrence и настройками напоминаний.
- `habit_logs`: дневной статус выполнения привычки (`full`, `micro`, `missed`).
- `notification_settings`: пользовательские настройки уведомлений и таймзона.
- `notifications`: планируемые задания напоминаний, связанные с привычками.
- `notification_logs`: журнал попыток отправки.
- `user_subscriptions`: тариф, даты действия и автопродление.
- `messages`: пул мотивационных сообщений.

## Основные связи

- У одного пользователя может быть много привычек.
- У одной привычки может быть много логов.
- У одного пользователя одна запись `notification_settings` и одна запись `user_subscriptions`.
- `notifications` может ссылаться на привычку, при этом действует ограничение одна запись на привычку.

## Ограничения целостности

- Уникальный email и в `users`, и в `pending_registrations`.
- Уникальная пара `(habit_id, date)` в `habit_logs`.
- Уникальный `habit_id` в `notifications` (одна активная строка расписания на идентичность привычки).
